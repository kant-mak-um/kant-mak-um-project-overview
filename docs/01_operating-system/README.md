
Operating System
This folder defines how work is actually done inside the Kant‑Mak‑Um ecosystem.
If 00_governance defines what must be true, this section defines how we act on it day‑to‑day and week‑to‑week.
The operating system is intentionally lightweight, repeatable, and forgiving. Its purpose is to maintain forward motion across many parallel efforts without burnout or loss of coherence.


Purpose of the Operating System
The operating system exists to answer five practical questions:

How priorities are set and revised
How work is planned at weekly and daily scales
How progress is reviewed and reset
How failures are handled without penalty
How learning, execution, and reflection coexist
This system is designed for long‑horizon work involving engineering, manufacturing, education, and open design.


Core Operating Principles
1. Dynamic Priorities, Stable Rules

Priorities may change weekly or even daily
Rules and boundaries change rarely
Re‑prioritization happens through review, not impulse


2. Time‑Boxed Focus
Work is organized into bounded blocks, not endless task lists.

Deep technical work
Production / commercialization
Governance and coordination
Learning and study
Finishing a block counts as success even if a task remains incomplete.


3. Auto‑Reset Discipline
Missed goals incur no penalty.
Every review cycle ends with:

A reset of priorities
A clean slate for the next cycle
No backlog guilt
This rule exists to support persistence over years, not heroics over days.


4. Separation of Thinking Modes
Different kinds of work are not mixed casually:

Strategy and governance
Execution
Learning
Reflection
The operating system exists to prevent mode confusion, which is a major source of inefficiency and frustration.


Planning Cadence
Weekly Cycle (Primary)
The week is the fundamental planning unit.
A typical weekly cycle includes:

Setting 1–3 primary outcomes
Identifying constraints and risks
Allocating time blocks by category
Reviewing results at week’s end
Resetting without penalty
The week is long enough to produce real progress and short enough to adapt.


Daily Execution (Secondary)
Daily planning is intentionally minimal:

Select which weekly outcomes to advance
Choose the appropriate work mode
Execute within a bounded time block
