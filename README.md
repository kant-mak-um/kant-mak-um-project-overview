# Kant‑Mak‑Um Project Overview

This repository is the **coordination core** for the Kant‑Mak‑Um ecosystem.

It defines the **governance, structure, and interfaces** that allow a large,
open, multi‑repo system to function coherently across:

- Open industrial design (public CAD & configurations)
- Closed manufacturing and tooling IP
- Multiple product companies and brands
- Academic programs (MS‑ECE and ME‑EM, CU Boulder)
- Long‑term scaling toward advanced manufacturing and aerospace applications

This repository intentionally contains **no product designs** and **no manufacturing processes**.

---

## Purpose

The purpose of this repository is to answer one question clearly:

> **How do all Kant‑Mak‑Um repositories relate to each other?**

It serves as the **single source of truth** for:
- Repository roles and boundaries
- Open vs closed IP separation
- Naming conventions
- Coordination rules
- Academic integration
- Change and decision discipline

This repo is **authoritative** for structure, not execution.

---

## What This Repository Contains

- Governance documents and non‑negotiable rules
- GitHub ecosystem structure (repo skeleton and registry)
- Operating rhythm (weekly dashboards, reviews, resets)
- Academic coordination (MS‑ECE and ME‑EM integration)
- Learning and tooling standards
- Cross‑project concurrency rules

---

## What This Repository Does *Not* Contain

This repository explicitly does **not** contain:

- Product CAD or configurations  
- Manufacturing processes or tooling  
- ORCAS motor internals  
- Coconino arm internals  
- Proprietary AFP methods  
- Brand‑specific execution artifacts  

Those live in their **respective repositories**, under their own licenses
and access controls.

---

## Ecosystem Position

This repository sits at the **center** of a tiered ecosystem:

- **Coordination Core** (this repo)
- **Open Design Commons** (public CAD & configurations)
- **Shared Components** (open, reusable parts)
- **Manufacturing Core** (private, proprietary)
- **Product Companies** (public designs, commercial execution)

If there is ever ambiguity about where something belongs,
**this repository defines the answer**.

---
